# ⚡ AWS Lambda (Serverless Compute)

## 📚 Complete Lambda Guide (Beginner → Intermediate → Practical)
### ⚡ Run code without managing servers

---

## 🔹 What is Lambda?
AWS Lambda is a serverless compute service that allows you to run code without provisioning or managing servers.

**👉 You only upload code and Lambda handles:**
* Infrastructure
* Scaling
* Execution

---

## 🔹 Why Lambda is Needed?
* No server management
* Automatic scaling
* Pay only for execution time
* Ideal for event-driven architectures

---

### 🔹 Key Concepts

#### 🔹 Function
A piece of code that runs in Lambda.

#### 🔹 Event
Trigger that invokes the function.

#### 🔹 Execution Role
IAM role that defines permissions for Lambda.

#### 🔹 Runtime
Environment in which code runs (Node.js, Python, Java, etc.)

---

## 🔹 How Lambda Works
1. Event occurs (trigger)
2. Lambda function is invoked
3. Code executes
4. Result is returned

---

## 🔥 Event Sources (Triggers)
* S3 (file upload)
* API Gateway (HTTP request)
* CloudWatch (scheduled events)
* DynamoDB
* SNS/SQS

---

## 🔥 Execution Model
* Stateless execution
* Each request runs independently
* Temporary environment

---

## 🔥 Scaling
* Automatically scales based on incoming requests
* No manual intervention required

---

## 🔐 Security

#### 🔹 IAM Role
Defines what Lambda can access

#### 🔹 Resource Policies
Control who can invoke Lambda

---

### 🔄 Lambda Layers
* Used to share code and dependencies
* Reusable across multiple functions

---

### 🔁 Monitoring

#### 🔹 CloudWatch Logs
Stores logs generated by Lambda

#### 🔹 Metrics
* Invocations
* Errors
* Duration

---

### 🌐 Integration
Lambda integrates with multiple AWS services:
* S3
* API Gateway
* DynamoDB
* EventBridge

---

## 🔍 Real Use Cases
* File processing (image resize)
* Backend APIs
* Automation tasks
* Scheduled jobs (cron)

---

## ⚠️ Common Mistakes
* Long-running tasks
* Large deployment packages
* Not handling timeouts
* Over-permission IAM roles

---

## 🔒 Best Practices
* Keep functions small and focused
* Use environment variables
* Optimize memory and timeout
* Use least privilege IAM roles
* Monitor logs regularly

## 🧪 Practical Steps

### Create Lambda Function
1. Go to Lambda Dashboard
2. Click Create Function
3. Choose runtime
4. Write/upload code
5. Configure role
6. Deploy

### Test Function
1. Create test event
2. Run function
3. Check output and logs
